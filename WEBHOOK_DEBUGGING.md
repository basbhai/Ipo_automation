# Debugging GitHub Actions Webhook Integration

## Current Issue
- Logs endpoint returning 404: No logs being stored
- Results endpoint returning 404: No results being stored
- This means Python script hasn't sent any data to the webhooks

## Root Cause Analysis

The problem is likely one of these:

### 1. ❓ Webhook URLs Not Being Passed to Python Script
GitHub Actions environment variables might not be accessible the way we expect.

**Check**: Look at GitHub Actions logs for the step "Run IPO application script"
- Do you see the env vars being printed?
- Add debug output to see if they're set

### 2. ❓ Python Script Crashing Before Sending Logs
If the script crashes during initialization, no logs will be sent.

**Check**: In GitHub Actions logs, look for:
- ImportError messages
- JSON parsing errors
- Any exception before "Processing X account(s)..."

### 3. ❓ Webhook URLs Are Invalid
If the URLs generated by dispatch are wrong, logs will fail silently.

**Check**: In Dispatch step logs, verify:
- `appUrl` is correct (should be your Vercel URL)
- `logsWebhookUrl` is correct
- `resultsWebhookUrl` is correct

## Next Steps to Debug

### Step 1: Check Dispatch Logs
1. Go to Vercel dashboard
2. Find your project
3. Go to Deployments
4. Click on current deployment
5. Go to "Logs" tab
6. Look for `[Dispatch] App URL:` and `[Dispatch] Webhook URLs:`
7. **Verify the URLs are correct** (should start with `https://your-domain.com`)

### Step 2: Check GitHub Actions Logs
1. Go to your GitHub repo
2. Click "Actions"
3. Find the "Mero Share IPO Bulk Applicator" workflow
4. Click the recent run
5. Click "Run IPO application script" step
6. Look for:
   - `=== Webhook Configuration ===`
   - `LOGS_WEBHOOK_URL: SET` or `LOGS_WEBHOOK_URL: NOT SET`
   - Any errors before "Processing X account(s)..."

### Step 3: Manual Test
Try triggering manually from GitHub:

```bash
curl -X POST https://api.github.com/repos/YOUR_OWNER/YOUR_REPO/dispatches \
  -H "Authorization: token YOUR_GITHUB_PAT" \
  -H "Accept: application/vnd.github.v3+json" \
  -d '{
    "event_type": "trigger_ipo_bot",
    "client_payload": {
      "jobId": "manual_test_1",
      "accounts": "[{\"dp\":\"TEST\",\"username\":\"test@example.com\",\"password\":\"test\",\"pin\":\"1234\",\"crn\":\"1234567890\",\"units\":\"100\"}]",
      "logsWebhookUrl": "https://your-app.vercel.app/api/logs",
      "resultsWebhookUrl": "https://your-app.vercel.app/api/results"
    }
  }'
```

This sends the webhook URLs explicitly to see if the Python script receives them.

## Common Issues & Fixes

### Issue: Webhook URLs are `http://localhost:3000`
**Fix**: This means appUrl detection is failing. The dispatch code now logs the actual URL being used. Check Vercel logs.

### Issue: `LOGS_WEBHOOK_URL: NOT SET` in GitHub logs
**Fix**: The environment variables aren't being passed to Python script. Even though they're in the workflow file, they might not be accessible via `os.getenv()` the way we expect.

**Solution**: Might need to use different approach - environment variables might need to be set differently in GitHub Actions.

### Issue: JSON parse error in logs
**Fix**: The ACCOUNTS_JSON might be getting corrupted or incorrectly escaped. The dispatch code now logs the actual URL patterns.
